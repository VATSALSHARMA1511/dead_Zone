DEADZONE — Systems & Gameplay Changes Implemented
1. Player Velocity Tracking System
File Modified:
player.py
What Was Added

Added persistent movement velocity tracking:

self.vel_x
self.vel_y
Why

Previously:

player movement only changed position,
no stored velocity existed,
AI could not predict movement direction.

Now:

zombies can estimate future player movement,
movement systems become expandable,
enables interception AI and advanced dodge systems.
Changes Made
Added in __init__()
self.vel_x = 0.0
self.vel_y = 0.0
Modified _move()

Old:

self.x += nx * speed * dt
self.y += ny * speed * dt

New:

self.vel_x = nx * speed
self.vel_y = ny * speed

self.x += self.vel_x * dt
self.y += self.vel_y * dt
Added stop reset logic

When player stops moving:

self.vel_x = 0.0
self.vel_y = 0.0
Modified dash system

Updated _update_dash() so dash also updates velocity:

self.vel_x = self._dash_vx
self.vel_y = self._dash_vy
2. Predictive Zombie AI System
File Modified:
zombie.py
What Was Changed

Previously zombies:

directly chased current player position

Now zombies:

predict future player position

using velocity prediction.

Added Predictive Chase Logic

Old:

dx = player.x - self.x
dy = player.y - self.y

New:

prediction_strength = 0.28

target_x = player.x + player.vel_x * prediction_strength
target_y = player.y + player.vel_y * prediction_strength

dx = target_x - self.x
dy = target_y - self.y
Gameplay Result

Zombies now:

intercept movement,
pressure movement paths,
cut angles,
reduce infinite kiting.

This made gameplay:

movement-based instead of pure backpedal shooting
